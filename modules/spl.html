interface Iterator
      |  
      `--(Traversable)

Allows to scan the (key,element) pairs of a collection, one by one

Objects that implement this interface can also be used in the foreach() statement: 

	class MyCollection implements Iterator { ... }
	$c = new MyCollection();
	foreach($c as $k => $e){ ... }
The iterator keeps an internal cursor which is initially positioned over the first pair, if it exists. Use valid() to check if the cursor i currently over a pair, then use next() to move to the next pair. Example: 
	$c->rewind();
	while( $c->valid() ){
		$key = $c->key();
		$element = $c->current();
		// ...do something with $key, $element
		$c->next();
	}
The behaviour of this interface when the underlying collection gets changed in between the scanning with the iterator is undefined; results may depend on the implementation.
{
void rewind()

Moves the cursor to the first (key,element) pair
boolean valid()

Check if the cursor is on a valid element
mixed key()

Returns the key of the current (key,element) pair under the cursor

Prerequisite: the cursor must be over a valid element, or the behaviour is unspecified; implementations may throw an unchecked exception to help debugging programs.
mixed current()

Returns the element of the current (key,element) pair under the cursor

Prerequisite: the cursor must be over a valid element, or the behaviour is unspecified; implementations may throw an unchecked exception to help debugging programs.
void next()

Move the cursor to the next (key,element) pair, if any

If the cursor is already beyond the last pair, does nothing.
}

interface Serializable

Classes that implements this interface provide a custom serialization algorithm
{
string serialize()

Performs custom serialization

The standard serialize() function call this method to retrieve serialized data.

Return: Custom serialized data. You may return any string here that represents the state of this object and that allows to restore this state. The format of this string is specific of the implementation.
void unserialize(
        string $serialized)

Unserializes data from custom serialization

The standard unserialize() function detects if a custom serialization algorithm is used, then creates the object from its class and calls this method instead of the constructor. This method must then initialize the object out of the serialized data generated by the Serializable::serialize() method.

WARNING. Calling this method from user code may destroy the object overwriting arbitrary data.

Parameters:
$serialized   Serialized data as generated by the Serializable::serialize() method.

Throws:
  • unchecked RuntimeException    You may want to throw this exception is the serialized data passed are invalid. The official manual does not specify what to do in this case. The standard unserialize() function triggers an E_NOTICE, but trigger_error("",E_NOTICE) cannot be called here when errors are remapped into exceptions: in fact, a checked ErrorException would be thrown, but this would violate the contract of the interface as stated in the PHP manual.
}
